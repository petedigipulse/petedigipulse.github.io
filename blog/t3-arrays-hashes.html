Accessing Elements

  • array.first - Returns the first element in the array.
  • array.last - Returns the last element in the array.
  • array.take(3) - Returns the 1st n element (1,2,3,)
  • array.drop(3) - Opposite of take - Afer n element (4,5,6)

Obtaining Information About Array

  • array.length - Returns the mumber of elements
  • array.count - Returns the number if elements
  • array.size - Returns the number if elements
  • array.empty? - Returns false
  • array.include? - Is a boolean ie ('not_here') => false

Adding Items To An Array

  • array.push or << - Items can be added to the end of the array
  • array.unshift - Add to the beginning of the array
  • array.insert - (3, 'apples') - Can insert multiple values

Hashes

Hashes are a great method to use if you need to take multiple sets of data. Hashes provide a key and a value for each set. Hashes, unlike arrays provide unordered sets of data. Arrays orders the data by integers, whereas hashes does not have any particular order for the data but can eaily be referenced by their key or their value.

Hashes are usually recoginised by the {} signs and their elements are identified by as so: hash = {key => value}. A hash can be created in multiple ways ie Class type this_is_a = Hash.new()

Hashes can hold multi-dimensional or mixed elements like integers, strings, arrays etc

Some of the built in methods are:

  • hash.keys - Accessing the hash by keys
  • hash.values - Accessing the hash by values
  • hash.size - Return the number of elements
  • hash.to_a - Creates an array with hash elements
  • hash.clear - Sets hash back to {}
  • hash.index - Returns the values of the hash ie me.index('Peter') => "first_name"
  • hash.length - Returns the number of elements in the hash.

Hashes can also take symbols as keys. When assiging a value to a key. You would like so hash = {my_symbol: 'Peter'
